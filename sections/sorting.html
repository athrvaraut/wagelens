Algorithm 01 — Comparison-Based

Employee Salary Ranking

Sorting algorithms organize employee records by compensation. Visualize Bubble Sort, Selection Sort, and Quick Sort on a live payroll dataset in real-time.

⏱ Best: O(n log n)  |  Worst: O(n²)  |  Space: O(1)~O(n)

Bubble Sort

Repeatedly compares adjacent employees, swapping if out of order. Simple but slow for large payrolls.

O(n²) time | O(1) space

Selection Sort

Finds minimum salary in unsorted portion and places it at the beginning. Good for small HR datasets.

O(n²) time | O(1) space

Quick Sort

Picks a pivot salary, partitions around it, and recursively sorts. Industry standard for large payrolls.

O(n log n) avg | O(log n) space

Bubble Sort Pseudocode
for i = 0 to n-1:
  for j = 0 to n-i-1:
    if sal[j] > sal[j+1]:
      swap(sal[j], sal[j+1])
// O(n²) comparisons
Selection Sort Pseudocode
for i = 0 to n-1:
  minIdx = i
  for j = i+1 to n:
    if sal[j] < sal[minIdx]:
      minIdx = j
  swap(sal[i], sal[minIdx])
Quick Sort Pseudocode
quickSort(arr, lo, hi):
  if lo < hi:
    pi = partition(arr,lo,hi)
    quickSort(arr,lo,pi-1)
    quickSort(arr,pi+1,hi)
// Divide & Conquer
